# Figma 设计稿搜索机器人 - 设计文档

**日期**：2026-03-02
**状态**：已确认

---

## 背景

团队有多个产品线的 Figma 设计文件（PPIO/NOVITA/JIEKOU 等），同一功能可能有多个迭代版本分散在不同文件中。需要一个工具帮助团队成员快速找到设计稿链接。

## 需求

1. **入口**：飞书机器人/小助手
2. **交互**：分步引导（产品名称 → 文件类型 → 搜索页面/板块）
3. **数据**：多层级（产品组 → 文件 → 页面 → Frame/Section）
4. **排序**：按更新时间，最新在前
5. **权限**：飞书组织成员可用

## 技术方案：飞书多维表格 + 机器人

### 架构

```
┌─────────────────────────────────────────────────────────┐
│                     数据层                               │
│  Figma 文件 ──同步脚本──→ 飞书多维表格                    │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│                     交互层                               │
│  用户 ←──→ 飞书机器人 ←──查询──→ 飞书多维表格             │
└─────────────────────────────────────────────────────────┘
```

### 飞书多维表格结构

| 字段 | 类型 | 示例 |
|------|------|------|
| 产品名称 | 单选 | PPIO / NOVITA / JIEKOU |
| 文件类型 | 单选 | 设计系统 / 常规迭代 |
| 文件名 | 文本 | PPIO-Design-System |
| 页面名称 | 文本 | 首页 |
| 板块名称 | 文本 | 登录弹窗 |
| Figma 链接 | 链接 | https://figma.com/file/xxx?node-id=123 |
| 更新时间 | 日期 | 2026-03-01 |

### 飞书机器人交互流程

```
用户：@设计稿助手

机器人：请选择产品：
        [PPIO] [NOVITA] [JIEKOU] [设计系统]

用户：点击 [PPIO]

机器人：请选择文件类型：
        [设计系统] [常规迭代]

用户：点击 [常规迭代]

机器人：请输入要搜索的页面/板块关键词：

用户：登录

机器人：找到 3 个相关设计稿：
        1. 登录弹窗 v2 (2026-03-01) 🔗
        2. 登录页面 (2026-02-15) 🔗
        3. 登录流程 (2026-01-20) 🔗
```

### 组件清单

| 组件 | 说明 |
|------|------|
| 飞书多维表格 | 存储 Figma 文件索引 |
| 同步脚本 | Figma API → 多维表格（Python） |
| 飞书机器人 | 消息卡片交互 + 查询多维表格 |
