# AI 团队统一规则

> 所有 Agent 必须遵守的规则，确保协作一致性。

---

## 架构原则

```
┌─────────────────────────────────────────────────────┐
│                   统一规则层                         │
│              RULES.md（本文件）                      │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│                 小秘书（中枢）                        │
│         接收任务 → 分配 → 汇总 → 汇报                │
└─────────────────────────────────────────────────────┘
           ↙      ↓       ↓       ↓      ↘
      设计师   Figma    知识    测试    摄影师
              设计员   管理     QA
```

---

## 1. 汇报规范

### 所有 Agent 完成任务后，必须写入 hub.json

```json
{
  "from": "Agent名称",
  "time": "ISO时间戳",
  "type": "类型",
  "content": "简短描述",
  "data": { "可选的附加数据" }
}
```

### 消息类型（type）

| 类型 | 说明 | 示例 |
|------|------|------|
| `update` | 更新/完成任务 | 设计系统更新到 v1.0.4 |
| `alert` | 发现问题需要关注 | 检测到 3 处硬编码颜色 |
| `info` | 普通信息通知 | 今日新增 2 个知识条目 |
| `request` | 请求其他 Agent 协助 | 需要设计师确认 Token |

### 写入示例

```json
{
  "from": "测试QA",
  "time": "2026-03-05T14:30:00+08:00",
  "type": "alert",
  "content": "发现 3 处硬编码颜色，需要确认",
  "data": {
    "issues": [
      { "file": "index.html", "line": 23, "value": "#ff0000" },
      { "file": "style.css", "line": 45, "value": "#333" }
    ]
  }
}
```

## 2. 权限边界

### 各 Agent 职责范围

| Agent | 可以做 | 不可以做 |
|-------|--------|----------|
| **小秘书** | 汇总信息、分配任务、生成报告、推送飞书 | 直接修改设计/代码 |
| **设计师** | 更新 Design Tokens、维护映射表 | 推送 GitHub（需确认） |
| **Figma 设计员工** | 在 Figma 中创建/修改元素 | 修改本地代码文件 |
| **知识管理** | 添加知识到 Notion/本地 | 修改设计系统 |
| **测试QA** | 检测代码、生成报告 | 自动修改代码（需确认） |
| **摄影师** | 获取 Unsplash 数据、推送报告 | 修改其他 Agent 的文件 |

### 跨职责协作

当需要其他 Agent 的能力时：
1. 在 hub.json 中写入 `type: "request"` 的消息
2. 指定目标 Agent（`to` 字段）
3. 等待小秘书分配或目标 Agent 响应

## 3. 设计系统规范

### 三层映射必须一致

```
Figma Token 名称  →  CSS 变量名  →  工具类名
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
brand primary/1   →  --brand-primary-1  →  .bg-brand-primary-1
gray/text/1       →  --text-1           →  .text-1
space/16          →  --space-16         →  .p-16
```

### 禁止事项

| 禁止 | 原因 |
|------|------|
| 硬编码颜色值（如 `#1161fe`） | 无法实现主题切换 |
| 自创命名（如 `primary-blue`） | 破坏映射一致性 |
| 非规范间距（如 `15px`） | 应使用 `16px` 等规范值 |

## 4. 线上操作规范

### 必须等待转转确认

以下操作**禁止擅自执行**：

| 操作 | 说明 |
|------|------|
| `git push` | 推送代码到 GitHub |
| `npm publish` | 发布 npm 包 |
| 飞书群发消息 | 除了日报自动推送 |
| 删除/覆盖远程数据 | 任何破坏性操作 |

### 正确流程

```
完成本地操作 → 写入 hub.json → 小秘书汇总 → 告知转转 → 等待确认 → 执行线上操作
```

## 5. 任务流转规范

### 任务状态

| 状态 | 说明 |
|------|------|
| `pending` | 待处理 |
| `in_progress` | 进行中 |
| `completed` | 已完成 |
| `blocked` | 被阻塞（需要其他 Agent 或转转） |

### 任务分配

1. **用户直接指定**：转转明确说"让 XX 做"
2. **小秘书自动分配**：根据任务类型匹配 Agent 职责
3. **Agent 主动认领**：在 hub.json 中标记 `assignee`

## 6. 文件组织规范

### 共享文件（所有 Agent 可读写）

```
~/.claude/agents/
├── RULES.md          ← 本文件（只读参考）
├── hub.json          ← 协作中枢（读写）
└── hub-schema.md     ← 数据格式说明（只读参考）
```

### 私有文件（各 Agent 独立）

```
~/.claude/agents/<agent-name>/
├── README.md         ← Agent 说明文档
├── config.json       ← Agent 配置（可选）
└── ...               ← Agent 专属文件
```

---

## 7. 错误处理

### 遇到问题时

1. **记录错误**：在 hub.json 写入 `type: "alert"` 消息
2. **不要静默失败**：必须有明确的错误记录
3. **不要自动重试危险操作**：等待人工确认

### 错误消息格式

```json
{
  "from": "Agent名称",
  "time": "时间戳",
  "type": "alert",
  "content": "错误描述",
  "data": {
    "error": "详细错误信息",
    "context": "发生错误时的上下文"
  }
}
```
